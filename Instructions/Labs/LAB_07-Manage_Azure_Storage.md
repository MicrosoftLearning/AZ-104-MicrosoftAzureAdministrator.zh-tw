# 實驗室 07 - 管理 Azure 儲存體

## 實驗室簡介

在此實驗室中，您將瞭解如何建立 Azure Blob 和 Azure 檔案的記憶體帳戶。 您將瞭解如何設定及保護 Blob 容器。 您也會瞭解如何使用 儲存體 Browser 來設定及保護 Azure 檔案共用。 

此實驗室需要 Azure 訂用帳戶。 您的訂用帳戶類型可能會影響此實驗室中功能的可用性。 您可以變更區域，但步驟是使用 **美國**東部撰寫。

## 預估時間：50 分鐘

## 實驗案例

您的組織目前正在內部部署數據存放區中儲存數據。 這些檔案大多不會經常存取。 您想要將不常存取的檔案放在較低價格的儲存層中，以將記憶體成本降到最低。 您也計畫探索 Azure 儲存體提供的不同保護機制，包括網路存取、驗證、授權和複寫。 最後，您想要判斷 Azure 檔案儲存體 適合裝載內部部署檔案共用的程度。

## 互動式實驗室模擬

您可能會發現互動式實驗室模擬對本主題很有用。 模擬可讓您以自己的步調點選類似的案例。 互動式模擬和此實驗室之間有差異，但許多核心概念都相同。 不需要 Azure 訂用帳戶。 

+ [建立 Blob 記憶體](https://mslearn.cloudguides.com/en-us/guides/AZ-900%20Exam%20Guide%20-%20Azure%20Fundamentals%20Exercise%205)。 建立記憶體帳戶、管理 Blob 記憶體，以及監視記憶體活動。 
  
+ [管理 Azure 記憶體](https://mslabs.cloudguides.com/guides/AZ-104%20Exam%20Guide%20-%20Microsoft%20Azure%20Administrator%20Exercise%2011)。 建立記憶體帳戶並檢閱設定。 管理 Blob 記憶體容器。 設定記憶體網路功能。

## 工作 1：建立和設定記憶體帳戶。 

在這項工作中，您將建立及設定記憶體帳戶。 記憶體帳戶會使用異地備援記憶體，而且不會有公用存取權。 

1. 登入 **Azure 入口網站** - `https://portal.azure.com`。

1. 搜尋並選取 `Storage accounts`，然後按兩下 [ **+ 建立**]。

1. 在 [建立記憶體帳戶 **] 刀鋒視窗的 **[**基本] 索引**卷標上，指定下列設定（讓其他人保留預設值）：

    | 設定 | 值 |
    | --- | --- |
    | 訂用帳戶          | Azure 訂用帳戶的名稱  |
    | 資源群組        | **az104-rg7** （create new） |
    | 儲存體帳戶名稱  | 長度介於 3 到 24 個字元之間，由字母和數字組成的任何全域唯一名稱 |
    | 區域                | **(美國) 美國東部**  |
    | 效能           | **標準** （請注意 進階版 選項） |
    | 備援            | **異地備援記憶體** （請注意其他選項）|
    | 在區域可用性的情況下對數據進行讀取存取 | 核取方塊 |

>**你知道嗎？** 您應該針對大部分的應用程式使用標準效能層級。 針對企業或高效能應用程式使用 進階版 效能層級。 

1. 在 [ **進階** ] 索引標籤上，使用參考圖示來深入了解選擇。 採用預設值。 

1. 在 [**網络] 索引**標籤上，檢閱可用的選項，選取 [**停用公用存取並使用私人存取]。**

1. 檢閱 [ **數據保護** ] 索引標籤。注意 7 天是預設的虛刪除保留原則。 請注意，您可以啟用 Blob 版本控制。 接受預設值。

1. 檢閱 [ **加密]** 索引標籤。請注意其他安全性選項。 接受預設值。

1. 選取 [ **檢閱**]，等待驗證程式完成，然後按兩下 [ **建立**]。

1. 部署記憶體帳戶之後，請選取 **[移至資源**]。

1. 檢閱 [ **概觀]** 刀鋒視窗和可變更的其他設定。 這些是記憶體帳戶的全域設定。 請注意，記憶體帳戶可用於 Blob 容器、檔案共用、佇列和數據表。

1. 在 [安全性 + 網路] **** 區段中，選取 [網路****]。 請注意，公用網路存取已停用。

    + 將 **公用存取層級** 變更為 **[從選取的虛擬網路和IP位址**啟用]。
    + 在 [ **防火牆]** 區段中，核取 [新增用戶端 IP 位址] 的 **方塊。**
    + 請務必**儲存**您的變更。 
  
1. 在 [ **數據管理** ] 區段中，檢視 [ **備援]** 刀鋒視窗。 請注意主要和次要數據中心位置的相關信息。

1. 在 [數據管理] 區**段中，選取 **[生命週期管理**]，然後選取 [**新增規則**]。**

    + **將規則`Movetocool`命名**為 。 請注意，限制規則範圍的選項。
    
    + 在 [**基底 Blob] 索引標籤上**，*如果*上次修改了基底 Blob，`30 days`*則***移至非經常性存取記憶體**。 請注意您的其他選擇。 
    
    + 請注意，您可以設定其他條件。 完成探索時，請選取 **[新增** ]。

    ![移至非經常性存取規則條件的螢幕快照。](../media/az104-lab07-movetocool.png)

## 工作 2：建立和設定安全的 Blob 記憶體

在這項工作中，您將建立 Blob 容器並上傳映像。 Blob 容器是儲存非結構化數據的類似目錄結構。

### 建立 Blob 容器和以時間為基礎的保留原則

1. 繼續 Azure 入口網站，使用您的記憶體帳戶。

1. 在 [資料儲存體]**** 區段中，按一下 [容器]****。 

1. 按兩下**列設定****建立**容器：

    | 設定 | 值 |
    | --- | --- |
    | 名稱 | `data`  |
    | 公用存取層級 | 請注意，存取層級設定為私人 |

    ![建立容器的螢幕快照。](../media/az104-lab07-create-container.png)

1. 在您的容器上，捲動至最右邊的省略號 （...），選取 [ **存取原則**]。

1. 在 [ **固定 Blob 記憶體** ] 區域中，選取 [ **新增原則**]。

    | 設定 | 值 |
    | --- | --- |
    | 原則類型 | **以時間為基礎的保留**  |
    | 設定的保留期間 | `180` 天 |

1. 選取 [儲存]。

### 管理 Blob 上傳

1. 返回容器頁面，選取您的數據**** 容器，然後按兩下 [**上傳**]。

1. 在 [ **上傳 Blob]** 刀鋒視窗上，展開 [ **進階** ] 區段。

    >**注意**：找出要上傳的檔案。 這可以是任何類型的檔案，但小型檔案是最好的。 您可以從 AllFiles 目錄下載範例檔案。 

    | 設定 | 值 |
    | --- | --- |
    | 瀏覽檔案 | 新增您選取要上傳的檔案 |
    | 選取 [進階]**** | |
    | Blob 類型 | **區塊 Blob** |
    | 區塊大小 | **4 MiB** |
    | 存取層 | **熱**  （請注意其他選項） |
    | 上傳到資料夾 | `securitytest` |
    | 加密範圍 | 使用現有的預設容器範圍 |

1. 按一下 [上傳] 。

1. 確認您有新的資料夾，且您的檔案已上傳。 

1. 選取您的上傳檔案並檢閱選項，包括 **下載**、 **刪除**、 **變更層**和 **取得租用**。

1. 複製檔案 **URL** 並貼到新的 **Inprivate** 瀏覽視窗中。

1. 您應該會看到 XML 格式的訊息，指出 **ResourceNotFound** 或 **PublicAccessNotPermitted**。

    > **注意**：這是預期的結果，因為您建立的容器已將公用存取層級設定為 [私人 (無匿名存取)]。

### 設定 Blob 記憶體的有限存取

1. 選取您上傳的檔案，然後在 [ **產生 SAS** ] 索引標籤上。您也可以使用極右的省略號 （...） 。 指定下列設定（讓其他人保留預設值）：

    | 設定 | 值 |
    | --- | --- |
    | 簽署金鑰 | **金鑰 1** |
    | 權限 | **閱讀** （請注意您的其他選擇） |
    | 開始日期 | 昨天的日期 |
    | 開始時間 | 目前時間 |
    | 到期日 | 明天的日期 |
    | 過期時間 | 目前時間 |
    | 允許的 IP 位址 | 保留空白 |

1. 按一下 [產生 SAS 權杖與 URL]。

1. 將 **Blob SAS URL** 專案複製到剪貼簿。

1. 開啟另一個 InPrivate 瀏覽器視窗，並流覽至您在上一個步驟中複製的 Blob SAS URL。

    >**注意**：您應該能夠檢視檔案的內容。 

## 工作 3：建立及設定 Azure 檔案記憶體

在這項工作中，您將建立及設定 Azure 檔案共用。 您將使用 儲存體 Browser 來管理檔案共用。 

### 建立檔案共享並上傳檔案

1. 在 Azure 入口網站 中，流覽回記憶體帳戶，在 **[數據記憶體**] 區段中，按兩下 [**檔案分享**]。

1. 按下 **[+ 檔案分享**]，然後在 [**基本] 索引**標籤上，為檔案共用提供名稱 `share1` 

1. 請注意階層**** 選項。 保留預設 **交易優化**。
   
1. 移至 [**備份]** 索引標籤，並確定**未**核取 [啟用備份****]。 我們正在停用備份以簡化實驗室設定。

1. 按兩下 [ **檢閱 + 建立**]，然後按兩下 [ **建立**]。 等候檔案共用部署。

    ![建立檔案共享頁面的螢幕快照。](../media/az104-lab07-create-share.png)

### 探索 儲存體 Browser 並上傳檔案

1. 返回您的記憶體帳戶，然後選取 **[儲存體 Browser**]。 Azure 儲存體 Browser 是入口網站工具，可讓您快速檢視帳戶下的所有記憶體服務。

1. 選取 [ **檔案分享** ]，並確認您的 **share1** 目錄存在。

1. 選取您的 **share1** 目錄，並注意您可以 **+ 新增目錄**。 這可讓您建立資料夾結構。

1. 選取**上傳**。 流覽至您選擇的檔案，然後按兩下 [ **上傳**]。

    >**注意**：您可以在 儲存體 Browser 中檢視檔案共享及管理這些共用。 目前沒有任何限制。

### 限制記憶體帳戶的網路存取

1. 在入口網站中，搜尋並選取 [虛擬網路]****。

1. 選取 **+ 建立**。 選取您的資源群組。 並提供虛擬網路**的名稱**。 `vnet1`

1. 取得其他參數的預設值，選取 [檢閱 + 建立 **]，然後**選取 **[建立**]。

1. 等候虛擬網路部署，然後選取 **[移至資源**]。

1. 在 **[設定]** 區段中，選取 [**子網]** 刀鋒視窗。
    + 選取 [預設] 子網路。****
    + 在 [**服務端點]** 區段中，選擇 **[服務 **] 下拉式清單中的 [Microsoft.儲存體****]。
    + 請勿進行任何其他變更。    
    + 請務必**儲存**您的變更。 

1. 返回您的儲存體帳戶。

1. 在 [ **安全性 + 網络功能]** 區段中，選取 [ **網络]** 刀鋒視窗。

1. 選取 **[新增現有的虛擬網络** ]，然後選取 **[vnet1** ] 和 **[預設** 子網]，選取 [ **新增**]。

1. 在 [防火牆]** 區**段中，**刪除**您的計算機 IP 位址。 允許的流量應該只來自虛擬網路。 

1. 請務必**儲存**您的變更。

    >**注意：** 記憶體帳戶現在應該只能從您剛才建立的虛擬網路存取。 

1. **選取 儲存體 瀏覽器**並**重新**整理頁面。 流覽至您的檔案共享或 Blob 內容。  

    >**注意：** 您應該會收到未獲授權執行此作業*的訊息*。 您不是從虛擬網路連線。 這可能需要幾分鐘的時間才會生效。

## 清除您的資源

如果您使用自己的訂用 **帳戶** ，需要一分鐘的時間才能刪除實驗室資源。 這可確保資源可以釋出，並將成本降到最低。 刪除實驗室資源最簡單的方式是刪除實驗室資源群組。 

+ 在 Azure 入口網站 中，選取資源群組、選取 **[刪除資源群組**]、**輸入資源組名**，然後按兩下 [**刪除**]。
+ 使用 Azure PowerShell， `Remove-AzResourceGroup -Name resourceGroupName`。
+ 使用 CLI， `az group delete --name resourceGroupName`。

## 重要心得

恭喜您完成實驗室。 以下是此實驗室的主要外賣。 

+ Azure 記憶體帳戶包含您所有 Azure 儲存體 資料物件：Blob、檔案、佇列和數據表。 儲存體帳戶會為 Azure 儲存體資料提供唯一的命名空間，可透過 HTTP 或 HTTPS 從世界各地存取該命名空間。
+ Azure 記憶體提供數個備援模型，包括本地備援記憶體（LRS）、區域備援記憶體（ZRS）和異地備援記憶體（GRS）。 
+ Azure Blob 記憶體可讓您在 Microsoft 的數據記憶體平台上儲存大量的非結構化數據。 Blob 代表二進位大型物件，其包含影像和多媒體檔案等物件。
+ Azure 檔案 儲存體 提供結構化數據的共享記憶體。 數據可以組織在資料夾中。
+ 不可變儲存體可讓您以寫入一次、讀取多次 (WORM) 狀態來儲存資料。 不可變的記憶體原則可以是時間型或法律保留原則。
