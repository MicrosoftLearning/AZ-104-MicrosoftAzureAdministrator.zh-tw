# 實驗室 01 - 管理 Microsoft Entra ID 身分識別

## 實驗室簡介

這是 Azure 系統管理員系列實驗室中的第一個實驗室。 在此實驗室中，您將了解使用者和群組。 使用者和群組是身分識別解決方案的基本建置組塊。 

此實驗室需要 Azure 訂用帳戶。 您的訂用帳戶類型可能會影響此實驗室中的功能可用性。 您可以變更區域，但這些步驟是使用**美國東部**作為區域來撰寫。 


## 預估時間：30 分鐘

## 實驗案例

貴組織正在建置新的實驗室環境，以進行進入生產階段前的應用程式和服務測試。  您聘請一些工程師來管理實驗室環境，包括虛擬機器。 為了允許工程師使用 Microsoft Entra ID 進行驗證，您必須負責佈建使用者和群組。 若要將系統管理額外負荷降到最低，群組的成員資格應該根據職稱自動更新。 

## 互動式實驗室模擬

此實驗室使用互動式實驗室模擬。 模擬可讓您以自己的步調點選類似的案例。 互動式模擬與此實驗室之間有所差異，但許多核心概念都相同。 不需要 Azure 訂用帳戶。

>**注意：** 正在更新此模擬。 Microsoft Entra ID 是 Azure Active Directory (Azure AD) 的新名稱。 

+ [管理 Entra ID 識別](https://mslabs.cloudguides.com/guides/AZ-104%20Exam%20Guide%20-%20Microsoft%20Azure%20Administrator%20Exercise%201)。 建立及設定使用者並指派給群組。 建立 Azure 建立並管理來賓帳戶。

## 工作 1：建立及設定使用者帳戶

在此工作中，您將建立及設定使用者帳戶。 使用者帳戶會儲存使用者資料，例如名稱、部門、位置和連絡人資訊。

1. 登入 **Azure 入口網站** - `https://portal.azure.com`。

1. 若要繼續進行入口網站，請在 [歡迎使用 Azure** 啟動顯示畫面] 上**選取 [**取消**]。 

    >**注意：** 所有實驗室都會使用 Azure 入口網站。 如果您不熟悉 Azure，請搜尋並選取 `Quickstart Center`。 花幾分鐘的時間觀看**在 Azure 入口網站中開始使用**影片。 即使您之前曾使用過入口網站，您也會找到瀏覽及自訂介面的一些秘訣和訣竅。
    
1. 搜尋並選取 `Microsoft Entra ID`。 Microsoft Entra ID 是 Azure 的雲端式身分識別與存取管理解決方案。 花幾分鐘的時間熟悉左側窗格中所列的一些功能。 

1. 選取 [概觀]**** 刀鋒視窗，然後選取 [管理租用戶]**** 索引標籤。 

    >**您知道嗎？** 租用戶是包含帳戶和群組的特定 Microsoft Entra ID 執行個體。 根據您的情況，您可以建立更多租用戶，並在租用戶之間進行**切換**。 

1. **在瀏覽器中按回或選取階層連結功能表中的選項，以返回 Entra ID** 頁面。

1. 選取 [ **授權**]。 您可以從這裡購買授權、管理您擁有的授權，以及將授權指派給使用者和群組。 選取 [授權功能]**** 以查看可用的功能。
   
### 建立新使用者

1. 選取 [使用者]****，然後在 [新增使用者]**** 下拉式清單中選取 [建立新使用者]****。 

1. 使用下列設定建立新使用者 (將其他設定保持預設值)。 在 [屬性]**** 索引標籤上，請注意使用者帳戶中所包含所有不同類型的資訊。 

    | 設定 | 值 |
    | --- | --- |
    | 使用者主體名稱 | `az104-user1` |
    | Display name | `az104-user1` |
    | 自動產生密碼 | **checked** |
    | 帳戶已啟用 | **checked** |
    | 職稱 (屬性索引標籤) | `IT Lab Administrator` |
    | 部門 (屬性索引標籤) | `IT` |
    | 使用位置 (屬性索引標籤) | **美國** |

1. 完成檢閱之後，請選取 [檢閱 + 建立]******，然後選取 [建立]**。

1. 重新整理頁面，並確認已建立新使用者。 

### 邀請外部使用者

1. 在 [新增使用者]**** 下拉式清單中，選取 [邀請外部使用者]****。 

    | 設定 | 值 |
    | --- | --- |
    | 電子郵件 | 您的電子郵件地址 |
    | Display name | 您的姓名 |
    | 傳送邀請訊息 | **核取方塊** |
    | 訊息 | `Welcome to Azure and our group project` |

1. 移至 [屬性]**** 索引標籤。完成基本資訊，包括這些欄位。 

    | 設定 | 值 |
    | --- | --- |
    | 職稱  | `IT Lab Administrator` |
    | 部門  | `IT` |
    | 使用位置 (屬性索引標籤) | **美國** |

1. 選取 [檢閱 + 邀請]****，然後選取 [邀請]****。

1. **重新整理**頁面，並確認已建立受邀的使用者。 您應該很快就會收到邀請電子郵件。 

    >**注意：** 您不太可能個別建立使用者帳戶。 您知道貴組織規劃建立及管理使用者帳戶的方式嗎？
    
## 工作 2：建立群組並新增成員

在這項工作中，您會建立群組帳戶。 群組帳戶可以包含使用者帳戶或裝置。 以下是將成員指派給群組的兩種基本方式：靜態和動態。 靜態群組需要系統管理員手動新增及移除成員。  動態群組會根據使用者帳戶或裝置的屬性自動更新。 例如，職稱。 

1. 在 Azure 入口網站中，搜尋並選取 `Groups`。

1. 花一分鐘的時間熟悉左側窗格中的群組設定。

   + **到期**可讓您以天為單位設定群組存留期。 之後，擁有者必須更新群組。
   + **命名原則**可讓您設定封鎖字組，並將前置詞或後綴新增至群組名稱。

1. 在 [所有群組]**** 刀鋒視窗中，選取 [+ 新增群組]****，然後建立新的群組。     

    | 設定 | 值 |
    | --- | --- |
    | 群組類型 | **安全性** |
    | 群組名稱 | `IT Lab Administrators` |
    | 群組描述 | `Administrators that manage the IT lab` |
    | 成員資格類型 | **已指派** |

    >**注意**：動態成員資格需要 Entra ID Premium P1 或 P2 授權。 如果有其他**成員資格類型**可供使用，選項會顯示在下拉式清單中。 
    
    ![建立指派群組的螢幕擷取畫面。](../media/az104-lab01-create-assigned-group.png)

1. 選取 [未選取任何擁有者]****。

1. 在 [ **新增擁有者** ] 頁面中，搜尋並 **選取** 您自己（顯示在右上角）作為擁有者。 請注意，您可以有多個擁有者。 

1. 選取 [未選取任何成員]****。

1. 在 [新增成員]**** 窗格中，搜尋並**選取** **az104-user1**，以及您邀請的**來賓使用者**。 將這兩個使用者新增至群組。 

1. 選取 [建立]**** 以部署群組。

1. [重新整理]**** 頁面，並確定已建立您的群組。

1. 選取新的群組，並檢閱 [成員]**** 和 [擁有者]**** 資訊。

>**注意：** 您可能會管理大量的群組。 貴組織是否打算建立群組和新增成員？
   
## 清除您的資源

如果您使用**自己的訂用帳戶**，請花點時間刪除實驗室資源。 如此可確保釋出資源，並將成本降到最低。 刪除實驗室資源的最簡單方式是刪除實驗室資源群組。 請注意，這不會移除您建立的任何 Entra ID 使用者或群組。 

+ 在 Azure 入口網站中，選取資源群組，選取 [刪除資源群組]****，[輸入資源群組名稱]****，然後按一下 [刪除]****。
+ 使用 Azure PowerShell `Remove-AzResourceGroup -Name resourceGroupName`。
+ 使用 CLI `az group delete --name resourceGroupName`。

## 重要心得

恭喜您完成此實驗室。 以下是此實驗室的一些主要重點：

+ 租用戶代表貴組織，並協助您管理內部和外部使用者的特定 Microsoft 雲端服務執行個體。
+ Microsoft Entra ID 具有使用者和來賓帳戶。 每個帳戶都有特定的存取層級，其專屬於預計要完成的工作範圍。
+ 群組會將相關使用者或裝置結合在一起。 有兩種類型的群組，包括安全性和 Microsoft 365。
+ 群組成員資格可以靜態或動態方式進行指派。
